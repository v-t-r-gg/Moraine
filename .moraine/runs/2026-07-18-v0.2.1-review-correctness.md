# Review correctness and safe persistence (v0.2.1)

## Objective

Ensure run-level review decisions can only be recorded against the exact persisted Markdown revision, and ensure concurrent or interrupted metadata writes cannot silently lose or corrupt ledger state.

## Scope

- Decisions only against saved Markdown (GUI gate + expected content hash).
- Revision conflict errors for Save and Decide.
- Per-document ledger lock with re-read-after-lock.
- Safe temp-file replacement without truncate fallback.
- Read-only `moraine status` and explicit `moraine init`.
- Deterministic legacy `.comments.json` migration (archive rename).

## Out of scope

- Annotation reanchoring and operation-based mutations (v0.3).
- Evidence capture, review inbox, auth, Git integrations.
- Full three-way merge.
- Package version bump to 0.2.1 (workspace remains 0.1.0; use milestone tag).

## Actions taken

- Introduced `atomic` module: unique same-directory temp write, safe replace, per-sidecar flock.
- `record_decision` verifies on-disk hash before append.
- CLI: read-only status, `init`, decide with optional `--expected-hash`.
- GUI: disable decide while dirty; base hash on save/decide; external conflict handling.
- Legacy migration archives to `.comments.json.migrated`.
- Docs updated for correctness boundaries.
- Manual CLI verification script run on 2026-07-18 (see Tests).

## Decisions and rationale

- **Package version stays 0.1.0** for this merge: crate/UI version was never bumped with product milestones. Tag as `review-ledger-v0.2.1` (or similar) rather than `v0.2.1` until versions are aligned.
- **Status never creates files** so CI and inspection of agent repos stay non-mutating.
- **GUI open still ensures ledger** so human open continues to assign a stable run id deliberately.
- **Full-list comment replace** remains host semantics; concurrent decision appends are locked. Full-list LWW for annotations deferred to v0.3 issue 1.

## Tests and evidence

Automated:


Manual CLI (2026-07-18):

```bash
# status read-only
rm -f /tmp/test.md.moraine.json
printf '# Test\n' > /tmp/test.md
moraine status /tmp/test.md --json
# run.initialized: false; no sidecar created

moraine init /tmp/test.md --json
# sidecar created; stable run id

HASH=$(sha256sum /tmp/test.md | cut -d' ' -f1)
moraine decide /tmp/test.md --decision approved --reviewer "Manual verification" --expected-hash "$HASH" --json
# reviewState approved, decisionCurrent true

printf '\nChanged\n' >> /tmp/test.md
moraine status /tmp/test.md --json
# reviewState stale, decisionCurrent false, decision history retained

# legacy: status does not migrate; init migrates once to .comments.json.migrated
```

## Known limitations

- Annotation host save still replaces the full comment list (last complete snapshot wins across processes).
- Windows replace uses move-aside sequence (short race window).
- Directory fsync is best-effort; not a durability certification.
- GUI adversarial paths require human desktop verification before merge.
