# Switch project license to Apache-2.0

## Objective

Ensure Moraine is licensed under **Apache-2.0 only** (remove dual MIT OR Apache-2.0).

## Scope

- Workspace and npm package license metadata
- Top-level license files
- README license section
- Root `package-lock.json` project license field only

## Out of scope

- Changing third-party dependency licenses
- Annotation anchoring, evidence, inbox, collab hardening
- Merge, tag, or human Moraine approval decision
- Broad refactors or product features

## Decisions and rationale

- Prefer a single canonical `LICENSE` file with the Apache License 2.0 text (standard GitHub/SPDX practice) rather than retaining dual `LICENSE-MIT` + `LICENSE-APACHE`.
- Set Cargo workspace and npm package `license` fields to `Apache-2.0`.
- Do not rewrite third-party license fields in `package-lock.json` (those packages remain dual-licensed by their upstreams).
- No product code changes; licensing metadata and files only.
